## 👋 О проекте (Зачем мне этот код?)

Привет! Если ты когда-нибудь чувствовал, что тонешь в потоке информации, теряешь гениальные идеи или долго не можешь принять сложное решение — добро пожаловать. Я написал этот API, чтобы решить проблему **информационного перегруза**.

**Second Brain API** — это не просто очередная "тудушка" или блокнот. Это полноценный бэкенд для твоего личного "второго мозга". Ты закидываешь сюда свои мысли, статьи, рецепты или идеи для стартапов, а под капотом происходит магия:
1. Нейросеть (Google Gemini) читает твои записи и переводит их в математические вектора (Embeddings).
2. Очередь задач (Celery + Redis) делает это в фоне, поэтому API работает молниеносно, без зависаний.
3. Векторная база данных (PostgreSQL + pgvector) бережно хранит смысл твоих текстов.
4. Когда у тебя возникает вопрос, ты задаешь его ИИ-Агенту, и он отвечает тебе **строго на основе твоих личных заметок** (архитектура RAG). Никаких галлюцинаций из интернета!

А еще я добавил сюда математический **Механизм принятия решений (Decision Engine)**. Больше никаких мук выбора "Какой фреймворк учить?" или "Куда поехать в отпуск?". Система сама высчитает идеальный вариант на основе твоих же приоритетов по взвешенным критериям.

## ✨ Главные фичи (Под капотом)

*   ⚡ **Сверхбыстрый API:** Написано на FastAPI с полной асинхронностью (`async/await`).
*   🔒 **Приватность превыше всего:** Строгая изоляция пользователей на уровне БД. Нейросеть физически не может "подглядеть" контекст чужих заметок. JWT-авторизация в комплекте.
*   🤖 **AI RAG Agent:** Семантический векторный поиск. ИИ не просто ищет по словам, он ищет *по смыслу*. (Используются модели `gemini-embedding-001` и `gemini-2.5-flash`).
*   🏋️ **Фоновая магия (Background Workers):** Походы к ИИ занимают время. Чтобы клиент не ждал, векторизация делегируется в Celery и Redis.
*   ⚖️ **Decision Engine:** Алгоритм принятия решений с использованием weighted scoring (оценки с учетом веса критериев).
*   🐳 **Полностью в Docker:** Настроен `docker-compose`, который поднимает API, базу данных (уже с активированным `pgvector`), воркер и редис буквально одной командой.

## 🚀 Быстрый старт (Запусти за 2 минуты)

Я постарался сделать процесс развертывания максимально безболезненным. Никаких танцев с бубном.

### 1. Клонирование и настройка
```bash
git clone https://github.com/твое_имя/second-brain-api.git
cd second-brain-api
```

Создай файл `.env` в корне проекта (или скопируй пример, если он есть) и обязательно вставь свой ключ от Google Gemini:
```env
POSTGRES_USER=second_brain_user
POSTGRES_PASSWORD=supersecret
POSTGRES_DB=second_brain
SECRET_KEY=supersecretkey_change_me_in_production
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=60
REFRESH_TOKEN_EXPIRE_DAYS=7
GEMINI_API_KEY=твой_ключ_здесь_без_пробелов_до_и_после_равно
```

### 2. Магия Docker Compose
Просто выполни эту команду. Docker сам скачает нужные базы, соберет наш FastAPI образ и запустит воркеры.
```bash
docker-compose up -d --build
```

### 3. Накатываем миграции (Один раз!)
Поскольку это первый запуск, нам нужно создать таблицы в базе данных. Выполни Alembic внутри контейнера API:
```bash
docker exec -it second_brain_api alembic upgrade head
```

Всё! API доступен по адресу `http://localhost:8000`.

## 🎮 Как этим пользоваться? (Документация)

FastAPI прекрасен тем, что сам генерирует красивую документацию. Просто открой в браузере:
👉 **[http://localhost:8000/docs](http://localhost:8000/docs)**

Там можно:
1. Создать пользователя (`POST /auth/register`)
2. Залогиниться и нажать кнопку **Authorize** сверху справа (вводим username/password).
3. Создать пару заметок (`POST /brain/`).
4. Подождать 2-3 секунды, пока Celery сделает вектора.
5. Написать вопрос в `POST /ai/chat` и наслаждаться тем, как ИИ отвечает по твоим текстам.

## 🏗️ Структура проекта (Для гиков)

Я старался придерживаться чистой архитектуры. Без фанатизма, но чтобы код было приятно читать через полгода:

```text
app/
├── api/          # Роутеры (эндпоинты). Здесь только логика приема и отдачи запросов.
├── core/         # Конфиги, безопасность (JWT, хэширование), зависимости (get_db).
├── db/           # Модели SQLAlchemy. Описание таблиц базы данных.
├── schemas/      # Pydantic модели. Строгая валидация входящих и исходящих JSON.
├── services/     # Бизнес-логика (работа с ИИ, расчет решений и т.д.).
└── workers/      # Celery воркеры для фоновых задач.
```
